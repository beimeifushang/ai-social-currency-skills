# Proof Patterns

Use this reference to choose a demo format that works on vertical video.

## Before/After

Best for productivity claims.

- Show the old workflow in 3-5 seconds.
- Show the new workflow in 10-20 seconds.
- Keep inputs identical.
- End with the exact time, cost, or cognitive step removed.

## Side-by-Side Race

Best for agent, automation, and coding claims.

- Split the screen into human/tool/agent lanes.
- Use the same task and visible timer.
- Highlight quality differences, not only speed.
- Include failure or correction if it happens; it builds trust.

## X-Ray Trace

Best for explaining invisible systems.

- Show logs, planning steps, graph nodes, tool calls, memory updates, or retrieval chunks.
- Animate state changes slowly enough to read.
- Keep only 3-5 labels on screen at a time.

## Simulator

Best for systems thinking and product strategy.

- Give the viewer a visible input control.
- Show one surprising output change.
- Explain the rule behind the change.
- Avoid fake precision; label sample data clearly.

## Failure Theater

Best for skeptical or high-trust audiences.

- Start with the claim.
- Show the failure boundary honestly.
- Explain why the boundary exists.
- Close with the more precise, defensible claim.

## Vertical Recording Rules

- Compose for 9:16 first, then adapt to desktop screenshots.
- Make the moving element large and central.
- Use overlays as labels, not paragraphs.
- Keep UI text above 16px equivalent in the final recording.
- Design the final frame as a shareable screenshot.
