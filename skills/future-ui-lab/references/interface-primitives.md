# Interface Primitives

Use this reference when selecting a post-chat AI interface pattern.

## Agent Canvas

Best for workflows with multiple steps, tools, and visible state.

- Core object: task graph, deal board, research map, or campaign plan.
- Human control: approve, reroute, pause, inspect, edit node.
- Demo move: show the graph changing as the agent works.

## Live Object

Best when a document, profile, plan, model, or contract becomes interactive.

- Core object: the artifact itself.
- Human control: highlight a part and ask the AI to transform, explain, or simulate it.
- Demo move: show the object reacting to constraints.

## Ambient Inbox

Best for monitoring, triage, and assistant workflows.

- Core object: stream of signals.
- Human control: rules, priority, escalation, digest cadence.
- Demo move: show raw noise becoming ranked actions.

## Camera Layer

Best for physical-world or screen-understanding workflows.

- Core object: what the camera or screen sees.
- Human control: point, circle, approve, ask, capture.
- Demo move: overlay labels, next actions, and confidence.

## Command Rail

Best when users need speed inside existing software.

- Core object: side rail beside the current app.
- Human control: shortcuts, intent chips, review queue.
- Demo move: show one command moving across multiple apps.

## Simulation Cockpit

Best for strategy, pricing, operations, or planning.

- Core object: variables and scenario outputs.
- Human control: sliders, assumptions, counterfactuals.
- Demo move: change one input and show a surprising outcome.

## Memory Surface

Best for relationship, creator, sales, or personal operating systems.

- Core object: persistent memory graph.
- Human control: pin, forget, correct, merge, permission.
- Demo move: show memory improving a future decision while exposing audit controls.

## Pattern Selection Rule

Choose the primitive that makes the AI's invisible work most visible. If two patterns fit, pick the one with clearer human control and easier 9:16 demonstration.
