# Roast Rubric

Use this reference when scoring whether a product is truly AI-native.

## Dimensions

Rate each from 1 to 5.

### Workflow Ownership

- 1: AI writes text inside an unchanged workflow.
- 3: AI completes a meaningful subtask.
- 5: AI owns a recurring workflow loop with human approval.

### Model Leverage

- 1: Generic summarization or chat.
- 3: Uses reasoning, retrieval, multimodal input, agents, or tools in a specific way.
- 5: Product value clearly depends on model capability that old software could not mimic.

### Interface Transformation

- 1: Chatbot pasted onto an old dashboard.
- 3: AI changes navigation, state, or editing.
- 5: The interface is built around intent, objects, traces, or delegation.

### Data Flywheel

- 1: No proprietary data or learning loop.
- 3: Uses domain data or feedback to improve output.
- 5: The product gets better through unique workflow data, not just more prompts.

### Trust Boundary

- 1: No audit, approval, privacy, or error recovery.
- 3: Basic review and visible sources.
- 5: Clear permissions, traceability, rollback, and human-in-the-loop design.

### Distribution Wedge

- 1: Could be cloned as a feature by incumbents.
- 3: Has a niche audience or channel.
- 5: Has a sharp buyer, urgent trigger, and distribution loop.

## Verdict Labels

- AI theater: average below 2.0.
- AI-wrapped: average 2.0-2.9.
- AI-assisted: average 3.0-3.9.
- AI-native: average 4.0+.

## Roast Lines

- "This is a chatbot renting space in an old product."
- "The AI is doing copywriting, not workflow ownership."
- "The demo is not fake, but the product boundary is timid."
- "The native move is hiding in the exception flow."
- "The killer feature is not the answer; it is the audit trail."

## Reputation Guardrails

- Critique visible product choices, not motives.
- Do not claim fraud, fake customers, or fake metrics without evidence.
- Use "suggests" and "appears" when working from limited public information.
